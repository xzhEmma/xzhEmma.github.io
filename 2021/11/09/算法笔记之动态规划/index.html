动态规划

算法笔记
Publish Date:   2021-11-09
Update Date:   2021-11-14

问题形式：求最值

暴力递归 ：自顶向下（从所求出发）

–> 子问题重叠 (eg: 斐波那契数列 – 大量节点被重复计算 )–> 带有备忘录的递归

迭代：自底向上（从已知出发）

–>状态转移方程

动态规划 –> 状态转移方程 时间复杂度 –> 优化：备忘录 DPtable (迭代) –> 滚动数组

时间复杂度：O(2^n）–> O(n)

空间复杂度： O(n) –> O(1) 

def coinChange(coins:List[int],amount:int):
##备忘录
    memo = dict()
def dp(n):
##查找备忘录，以免重复计算
    if n in memo:return memo[n]
#base case
    if n==0:reutrn 0;
	if n<0:return -1;
#求最小值，所以初始化为正无穷
    res = float('INF')
	for coin in coins:
		subproblem = dp(n-coin)
        #子问题无解，跳过
        if subproblem ==-1:continue
		res = min(res,1+subproblem)
        ##记入备忘录
    memo[n] = res if res != float('INF') elae -1
    return memo[n]
return dp(amount)
#迭代
int coinChange(vector<int>& coins,int amount){
    //初始化
    vector<int>dp(amount+1,amount+1);
    //base case
    dp[0] = 0;
    for(int=0;i<dp.size();i++){
        for (int coin:coins){
            if(i-coin<0) continue;
            dp[i] = min(dp[i],1+dp[i-coin]);
        }
    }
    return (dp[amount]==amount+1)? -1:dp[amount];

结语：计算机解决问题唯一的方式就是穷举，而算法就是从思考“如何穷举“到“如何聪明的穷举”
